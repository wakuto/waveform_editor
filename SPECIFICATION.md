# Waveform Editor — 仕様書

## 1. プロジェクト概要

### 1.1 目的
デジタル回路のタイミング設計で使用する**波形（タイミングダイアグラム）**を、GUIで直感的に編集できるWebアプリケーション。

### 1.2 背景と動機

| 既存ツール | 長所 | 短所 |
| :--- | :--- | :--- |
| **Excel** | 自由にレイアウト可能 | 波形専用ツールではなく描きづらい。diffが見づらくgit管理に不向き |
| **WaveDrom** | テキストベースでgit管理に最適。専用ツールで高品質な出力 | テキスト編集とプレビューの対応が見づらい。記法の学習コストがある |

本ツールは WaveDromの長所（テキストベース・git管理可能）を活かしつつ、GUIによる直感的な編集体験を提供することで、両者の欠点を補う。

### 1.3 コンセプト
- フロントエンドのみで動作（サーバー不要、静的ファイルのみ）
- 内部データ形式は WaveDrom JSON をそのまま使用
- GUIで編集 → WaveDrom JSONとして保存 → gitで管理
- WaveDrom JSONのインポートにも対応し、既存資産を活用可能

## 2. 対象ユーザー
- デジタル回路（FPGA / ASIC）のRTL設計者
- インターフェースタイミングを設計・ドキュメント化するエンジニア

## 3. 技術要件

### 3.1 動作環境
- モダンブラウザ（Chrome, Firefox, Edge, Safari の最新2バージョン）
- デスクトップ想定（最小画面幅: 1024px）

### 3.2 技術スタック
- フロントエンドのみ（バックエンドなし）
- TypeScript + React（UI構築）
- Vite（ビルドツール）
- Zustand（状態管理）
- 自前SVGレンダリング（波形の描画と直接編集を統合するため、WaveDromライブラリは使わず独自実装）
- CSS Modules（スタイリング）

### 3.3 波形レンダリング方針

#### WaveDromライブラリを使用しない理由
WaveDromライブラリは波形の「表示」には優れているが、以下の理由から本アプリでは採用しない：
- **直接編集との統合が困難**: WaveDromが生成するSVGは読み取り専用であり、各波形セグメントにインタラクティブなイベントを付与する仕組みがない
- **レンダリングの制御が限定的**: セル単位のホバー/選択状態の視覚的フィードバック、カーソル表示、ドラッグ操作中のリアルタイム反映など、エディタに必要なUI要素をWaveDromの外から制御するのが難しい
- **再レンダリングコスト**: 1セルの変更のたびにWaveDrom全体を再レンダリングする必要があり、大規模波形でのインタラクション性能が劣化する

#### 自前SVGレンダリングの方針
- React SVGコンポーネントとして波形を描画
- 各タイムステップの波形セグメントが独立したインタラクティブなSVG要素
- WaveDromと同等の見た目（波形の形状・色・ラベル）を再現
- セル単位の仮想DOM差分更新により、部分的な再レンダリングで高速動作
- 将来的にWaveDromのSVG出力と比較して品質を検証可能

### 3.4 デプロイ
- 静的ファイルとしてホスティング可能（GitHub Pages, Netlify 等）
- オフラインでもローカルで `index.html` を開いて使用可能（目標）

## 4. 機能要件

### 4.1 波形のGUI編集（コア機能）

#### 4.1.1 信号の管理

| 機能 | 説明 |
| :--- | :--- |
| **信号の追加** | 新しい信号行を追加する |
| **信号の削除** | 既存の信号行を削除する |
| **信号の並べ替え** | ドラッグ＆ドロップで信号の順序を変更する |
| **信号名の編集** | 信号名をインライン編集する |
| **信号のグループ化** | WaveDromのグループ構文に対応（ラベル付きグループ） |

#### 4.1.2 波形の編集
各信号行はグリッド（タイムステップ）単位で状態を編集できる。

| 波形状態 | WaveDrom記法 | 説明 |
| :--- | :--- | :--- |
| **Low** | `0` | 論理0 |
| **High** | `1` | 論理1 |
| **Positive Edge Clock** | `p` | 立ち上がりエッジクロック |
| **Negative Edge Clock** | `n` | 立ち下がりエッジクロック |
| **High-Z** | `z` | ハイインピーダンス |
| **Undefined** | `x` | 不定値 |
| **Data (色付き)** | `=`, `2`〜`9` | データバス（ラベル付与可能） |
| **Gap** | `\|` | 波形の省略（ギャップ） |
| **Continue** | `.` | 前の状態を継続 |

**編集操作（レンダリングされた波形上で直接実行）:**
- **クリック**: 波形上のタイムステップ位置をクリックして、現在選択中のツール（状態）を適用。波形がその場で即座に変化する
- **ドラッグ**: 波形上を横方向にドラッグし、複数タイムステップに一括で状態を適用。**ドラッグ範囲の左端セルのみ選択ツールの値（`0`/`1`/`=` など）を設定し、それより右のセルはすべて継続 `.` を挿入する**ことで自然な波形継続を実現する
- **ドラッグ中のフィードバック**: ドラッグ操作中は以下の視覚的フィードバックを表示する
  - SVG のカーソルが `grabbing` に変化し、ドラッグ操作中であることを示す
  - 開始セル（value を書き込むセル）を **黄色の明るいオーバーレイ** でハイライト
  - 継続セル（`.` を書き込むセル）を **薄い黄色のオーバーレイ** でハイライト
  - 開始・継続セルの波形ラインを明度アップして編集対象を強調
- **ダブルクリック（選択モード時）**: データ状態（`=`, `2`〜`9`）のセルでデータラベルをインライン編集する。クリックしたセルの位置に直接テキスト入力欄が表示され（`<foreignObject>` によるSVG内入力）、Enter または外側クリックで確定、Escape でキャンセル。継続 `.` セルも親のデータセルとして扱い編集可能。
- **右クリック**: 波形領域で右クリックすると、ツールを選択するための円形メニュー（ラジアルメニュー）が即座に表示される。メニューの中心で右クリックを離すとキャンセルされる。
- **ホバー**: マウスカーソル位置のタイムステップをハイライト表示し、編集対象を視覚的に示す

#### 4.1.3 タイムステップの管理

| 機能 | 説明 |
| :--- | :--- |
| **タイムステップの挿入** | 挿入カーソル位置に `.`（継続）で埋めたサイクルを挿入する |
| **タイムステップの削除** | 選択中のサイクル範囲を全信号から削除する |
| **タイムステップのコピー** | 選択中の範囲を全信号分クリップボードにコピーする |
| **タイムステップのカット** | 選択中の範囲を全信号分カットし削除する |
| **タイムステップのペースト** | クリップボードの内容を挿入カーソル位置に挿入する |

#### 4.1.4 タイムステップ選択ツール

ツールバーの**選択ツール** (`S` キー、またはツールパレットの `▷` ボタン) を選択すると、タイムステップ軸（列方向）の操作モードになる。波形状態の書き込みは行わず、挿入カーソルの移動・サイクル範囲選択・構造操作（挿入・削除・コピー等）に特化する。

**挿入カーソル:**

挿入カーソルはペースト・ステップ挿入を行う位置を示す。サイクルとサイクルの**境界（クロックエッジ位置）**に縦線として表示される。

- 選択ツール有効時、常に波形キャンバス全体にまたがる縦線として表示する
- カーソル位置はサイクル境界インデックス（0 = 先頭サイクルの前、N = 末尾サイクルの後）で管理する
- **ステップヘッダーのサイクル境界部分をクリック**すると、最近傍の境界にカーソルが移動する
- **選択モード中に波形上をクリック**すると、クリックした x 座標から最近傍のサイクル境界にスナップしてカーソルが移動する
- 挿入カーソルは選択ツール以外のツールに切り替えても位置が保持される

**挿入カーソルの視覚表示:**
- キャンバス全体の高さにわたる**シアン色の縦線**（幅 2px 程度）をサイクル境界上に表示
- ホバー時: カーソル候補の境界位置を薄い縦線でプレビュー表示

**サイクル選択:**

カット・コピー・一括削除のために、サイクル単位での範囲選択が可能。選択したサイクル全体（全信号行にまたがる縦帯）を青みがかったオーバーレイでハイライトする。
また、特定の信号（行）のみを選択してコピー＆ペーストすることも可能。

| 操作 | 動作 |
| :--- | :--- |
| **ステップヘッダーのサイクル内部をクリック** | そのサイクルを全信号で単一選択 |
| **ステップヘッダーをドラッグ** | ドラッグ範囲のサイクルを全信号で選択（複数サイクル選択） |
| **Shift + クリック** | 現在の選択範囲から指定サイクルまで選択を拡張 |
| **選択モード中に波形上をドラッグ** | ドラッグした信号（行）のみのサイクル範囲を選択（単一信号選択） |

**選択範囲に対する操作（ツールバーボタン / キーボードショートカット）:**

| 操作 | ショートカット | 動作 |
| :--- | :--- | :--- |
| **挿入** | `Insert` | 挿入カーソル位置に `.`（継続）で埋めた 1 サイクルを挿入 |
| **削除** | `Delete` / `Backspace` | 選択範囲のサイクルを削除（全信号選択時は全信号から、単一信号選択時はその信号のみから削除）。※削除範囲の直後が継続 `.` の場合は元の状態を解決して復元する |
| **コピー** | `Ctrl+C` | 選択範囲のスナップショットをアプリ内クリップボードにコピー（全信号選択時は全信号分、単一信号選択時はその信号のみ） |
| **カット** | `Ctrl+X` | コピーして削除 |
| **ペースト** | `Ctrl+V` | 選択範囲がある場合は選択範囲を削除してペースト（上書きペースト）。選択範囲がない場合は挿入カーソル位置に挿入（挿入ペースト）。<br>・**全信号コピー時**: 信号インデックスを合わせてペースト（余分な信号は捨て、足りない信号は `.` で補完）。<br>・**単一信号コピー時**: 選択中の信号（または最後に操作した信号）の挿入カーソル位置にペースト。<br>※先頭が継続 `.` の場合は元の状態を解決してペーストする |

**ペースト時の継続（`.`）の解決:**
コピーした範囲の先頭が継続 `.` だった場合、ペースト先の直前の状態を意図せず継続してしまうのを防ぐため、ペースト時に適切な値に変換して挿入する。
具体的には、コピー元の波形においてその `.` が実際に表していた状態（直前の有効な値、例えば `0`, `1`, `x`, `z`, `=`, `p`, `n` など）を解決し、ペーストする波形の先頭をその値で上書きする。
例：`0` の継続（`.`）をコピーして、`1` の継続の後ろにペーストした場合、そのまま `.` を貼り付けると `1` が継続されてしまうため、ペーストする先頭の値を `0` に変換して挿入する。
ただし、ペースト先の直前の状態と、ペーストする値（解決済みの値）が同じ場合は、状態を変化させる必要がないため、先ほど追加した動作を行わず、継続 `.` を挿入する。

**ペースト後の継続（`.`）の復元:**
ペーストによって波形が挿入された結果、ペースト範囲の直後のセルが継続 `.` だった場合、その `.` が意図しない状態を継続してしまうことがある。
これを防ぐため、ペースト範囲の直後のセルが `.` であり、かつペーストによって直前の状態が変化した場合、その `.` をペースト前の元の状態（`0`, `1`, `=`, `x` など）に置き換えて復元する。
例：`1...` の2サイクル目（最初の `.` の位置）に `0` をペーストした場合、ペースト直後は `10..` となるが、直後の `.` が `0` を継続してしまうため、元の状態である `1` に置き換え、`101.` とする。これにより、ペースト範囲の外側の波形が変化しないようにする。

**削除後の継続（`.`）の復元:**
削除によって波形が詰められた結果、削除範囲の直後のセルが継続 `.` だった場合、その `.` が意図しない状態を継続してしまうことがある。
これを防ぐため、削除範囲の直後のセルが `.` だった場合、その `.` を削除前の元の状態（`0`, `1`, `=`, `x` など）に置き換えて復元する。
例：`101.` の `0` の部分（1サイクル）を選択して削除した場合、削除直後は `11.` となるが、直後の `.` が前の `1` を継続してしまうため、元の状態である `1` に置き換え、`111` とする。これにより、削除範囲の外側の波形が変化しないようにする。

### 4.2 インタラクティブ波形キャンバス（統合型エディタ）
編集とレンダリングを分離せず、レンダリングされた波形そのものが編集UIとなる統合型のアプローチを採る。

- 自前のReact SVGコンポーネントにより波形を描画
- 描画された波形の各タイムステップが直接クリック/ドラッグ可能
- 編集操作に対して波形が即座に視覚的に変化（リアルタイムフィードバック）
- ホバー時のハイライト、選択中ツールに応じたカーソル変化
- グリッドライン（タイムステップ境界）を薄く表示し、編集位置を明確に
- **ドラッグ中**: 操作範囲をオーバーレイ色でリアルタイムに視覚化（value セルは明るい黄色、継続セルは薄い黄色）。カーソルも `grabbing` に変更

#### 波形の描画仕様

| 波形状態 | SVG表現 |
| :--- | :--- |
| **Low (`0`)** | 下端の水平線 |
| **High (`1`)** | 上端の水平線 |
| **Posedge Clock (`p`)** | 立ち上がり→立ち下がりの矩形波 |
| **Negedge Clock (`n`)** | 立ち下がり→立ち上がりの矩形波 |
| **High-Z (`z`)** | 中央の水平線（破線） |
| **Undefined (`x`)** | 左端 `<`・右端 `>` のスパイク付き六角形領域（赤色ハッチング） |
| **Data (`=`, `2`〜`9`)** | 左端 `<`・右端 `>` のスパイク付き六角形領域（色付き塗りつぶし）＋中央ラベル |
| **Gap (`\|`)** | 波線による切断表示 |
| **Continue (`.`)** | 前の状態を境界線なしで延長 |

#### 継続（`.`）の描画ルール

- **`1.` `0.`**: 境界に区切り線なし・フラットに延長（`1 1` との違いが明確）
- **`11` `00`（明示的な同値繰り返し）**: セル境界で中間高さまでディップ／バンプして戻り、信号変化を可視化
- **`=.` Data継続**: 左端 `<` なし・右端 `>` なし（矩形塗りつぶし）で連続表示
- **`x.` Undefined継続**: 左端スパイクなし・矩形ハッチングで延長

#### 左右スパイクの制御ルール（x / Data 共通）

| セルの状況 | 左端 `<` | 右端 `>` | 塗りつぶし形状 |
| :--- | :--- | :--- | :--- |
| 単独セル（前後が異なる状態） | あり | あり | 六角形 |
| 先頭セル（次が `.`） | あり | なし | 左 `<` 付き五角形 |
| 継続セル（前が `.`、次は別状態） | なし | あり | 右 `>` 付き五角形 |
| 中間継続セル（前後ともに `.`） | なし | なし | 矩形 |

```text
│  │ ack    │ _________╱‾‾╲___   │  │   "signal":.. │
│  │ data   │ xxxx<D0><D1>xxxx   │  │  }             │
│  └────────┴────────────────────┘  │               │
│  [+信号追加]                        │               │
├─────────────────────────────────────┴───────────────┤
│  ステータスバー: カーソル位置、選択ツール、信号数等      │

```

### 5.2 ツールバー
- ファイル操作ボタン群（新規、開く、保存、SVG/PNGエクスポート）
- Undo / Redo ボタン
- **選択ツールボタン**: `▷ 選択`（タイムステップ列選択モード）
  - **挿入ボタン**: 選択範囲の前にタイムステップを挿入
  - **削除ボタン**: 選択範囲を削除
  - **コピー / カット / ペーストボタン**
- 波形状態ツールパレット: 現在のペンツールを選択
  - `0`(Low), `1`(High), `p`(PosClk), `n`(NegClk), `z`(Hi-Z), `x`(Undefined)
  - `=`(Data) + データ色セレクタ（`2`〜`9`に対応）
  - `.`(Continue), `|`(Gap)
- ズームコントロール

### 5.3 波形キャンバス（統合型エディタ）
アプリのメイン領域。レンダリングされた波形がそのまま編集UIとなる。

- **信号ラベル領域（左側固定）**
  - 信号名を表示（ダブルクリックでインライン編集）
  - ドラッグ＆ドロップで信号の並べ替え
  - 右クリックでコンテキストメニュー（削除、複製、グループ化 等）
- **波形描画領域（右側、横スクロール可能）**
  - 自前SVGによる波形レンダリング
  - 各タイムステップのセグメントがインタラクティブ（クリック/ドラッグ/ホバー対応）
  - グリッドライン（タイムステップ境界）を薄く表示
  - ヘッダー行にタイムステップ番号またはtick/tockラベルを表示
  - エッジリレーション（矢印）を波形上にオーバーレイ描画
- **信号追加ボタン（下部）**

#### 編集のインタラクション詳細

| 操作 | 動作 |
| :--- | :--- |
| **波形上をクリック** | 編集ツール: クリック位置のタイムステップに選択中ツールの状態を適用 / 選択ツール: 最近傍サイクル境界に挿入カーソルを移動 |
| **波形上をドラッグ** | 編集ツール: ドラッグ範囲に一括適用 / 選択ツール: サイクル範囲を選択 |
| **ステップヘッダーのサイクル境界をクリック** | 挿入カーソルをその境界位置に移動（選択ツール・編集ツール共通） |
| **ステップヘッダーのサイクル内部をクリック** | 選択ツール: そのサイクルを単一選択 |
| **ステップヘッダーをドラッグ** | 選択ツール: サイクル範囲を選択 |
| **ホバー** | 編集ツール: 対象タイムステップをハイライト / 選択ツール: カーソル候補境界位置をプレビュー表示 |
| **データセルをダブルクリック** | ラベル編集用インラインポップアップを表示 |
| **信号名をダブルクリック** | 信号名のインライン編集 |
| **信号名をドラッグ** | 信号の並び替え |
| **右クリック** | コンテキストメニュー（信号の削除/複製/グループ化 等） |
| **ホイールスクロール** | 横方向: タイムステップ方向スクロール |
| **Ctrl + ホイール** | ズームイン/アウト |

### 5.4 JSONエディタパネル（サイドパネル）
- 波形キャンバスの右側に表示
- 開閉トグルボタンで表示/非表示を切り替え（デフォルト: 非表示で波形を広く使える）
- シンタックスハイライト付きテキストエディタ
- 波形キャンバスと双方向同期

### 5.5 ステータスバー
- 挿入カーソル位置（サイクル境界インデックス）
- 選択中のサイクル範囲（選択がある場合）
- 選択中のツール名
- 信号数 × タイムステップ数

## 6. データモデル
内部データはWaveDrom JSON形式をそのまま使用する。

### 6.1 WaveDrom JSON 形式（例）
```json
{
  "signal": [
    { "name": "clk",   "wave": "p........" },
    { "name": "req",   "wave": "0.1..0..." },
    { "name": "ack",   "wave": "0...1..0." },
    { "name": "data",  "wave": "x...=.=.x", "data": ["D0", "D1"] },
    {},
    ["Bus Interface",
      { "name": "addr",  "wave": "x..=.=.x.", "data": ["A0", "A1"] },
      { "name": "wdata", "wave": "x...=.=.x", "data": ["W0", "W1"] }
    ]
  ],
  "edge": [
    "req~>ack", "ack~>data"
  ],
  "head": {
    "text": "Read Transaction"
  },
  "config": {
    "hscale": 2
  }
}
```

### 6.2 アプリ内部状態
```typescript
interface AppState {
  /** WaveDrom JSON データ（信号、エッジ、head/foot/config を含む） */
  waveformData: WaveDromData;

  /** Undo/Redo 用の履歴スタック */
  undoStack: WaveDromData[];
  redoStack: WaveDromData[];

  /** UI状態 */
  selectedTool: WaveTool;        // 現在選択中の編集ツール（'select' を含む）
  selectedSignal: number | null; // 選択中の信号インデックス
  jsonPanelVisible: boolean;     // JSONパネルの表示/非表示

  /** タイムステップ選択・カーソル（選択ツール用）*/
  insertCursor: number | null;                        // 挿入カーソル位置（サイクル境界インデックス: 0〜N）
  stepSelection: { from: number; to: number; signalIndex?: number } | null; // 選択中のサイクル範囲（signalIndex がある場合は単一信号選択）
  stepClipboard: StepClipboard | null;                // コピー/カット済みデータ
}
```

## 8. 非機能要件

| 項目 | 要件 |
| :--- | :--- |
| **パフォーマンス** | 100信号 × 50タイムステップでスムーズに動作 |
| **レスポンシブ** | デスクトップ画面（1024px以上）で最適表示 |
| **アクセシビリティ** | キーボード操作に対応。ツールチップで操作ガイド |
| **オフライン** | 一度読み込めばオフラインでも動作（Service Worker検討） |
| **データ永続化** | ブラウザの LocalStorage に自動保存（クラッシュ対策） |

## 11. 用語集

| 用語 | 説明 |
| :--- | :--- |
| **WaveDrom** | JavaScriptベースのデジタルタイミングダイアグラム描画エンジン（https://wavedrom.com/） |
| **タイムステップ** | 波形の最小時間単位。グリッドの1列に相当 |
| **信号（Signal）** | 1本の波形ライン。名前と波形状態の配列で構成される |
| **エッジリレーション** | 信号間の時間的な因果関係を示す矢印 |
| **hscale** | WaveDromの水平方向のスケーリング係数 |
