# Waveform Editor — 仕様書

## 1. プロジェクト概要

### 1.1 目的
デジタル回路のタイミング設計で使用する**波形（タイミングダイアグラム）**を、GUIで直感的に編集できるWebアプリケーション。

### 1.2 背景と動機

| 既存ツール | 長所 | 短所 |
| :--- | :--- | :--- |
| **Excel** | 自由にレイアウト可能 | 波形専用ツールではなく描きづらい。diffが見づらくgit管理に不向き |
| **WaveDrom** | テキストベースでgit管理に最適。専用ツールで高品質な出力 | テキスト編集とプレビューの対応が見づらい。記法の学習コストがある |

本ツールは WaveDromの長所（テキストベース・git管理可能）を活かしつつ、GUIによる直感的な編集体験を提供することで、両者の欠点を補う。

### 1.3 コンセプト
- フロントエンドのみで動作（サーバー不要、静的ファイルのみ）
- 内部データ形式は WaveDrom JSON をそのまま使用
- GUIで編集 → WaveDrom JSONとして保存 → gitで管理
- WaveDrom JSONのインポートにも対応し、既存資産を活用可能

## 2. 対象ユーザー
- デジタル回路（FPGA / ASIC）のRTL設計者
- インターフェースタイミングを設計・ドキュメント化するエンジニア

## 3. 技術要件

### 3.1 動作環境
- モダンブラウザ（Chrome, Firefox, Edge, Safari の最新2バージョン）
- デスクトップ想定（最小画面幅: 1024px）

### 3.2 技術スタック
- フロントエンドのみ（バックエンドなし）
- TypeScript + React（UI構築）
- Vite（ビルドツール）
- Zustand（状態管理）
- 自前SVGレンダリング（波形の描画と直接編集を統合するため、WaveDromライブラリは使わず独自実装）
- CSS Modules（スタイリング）

### 3.3 波形レンダリング方針

#### WaveDromライブラリを使用しない理由
WaveDromライブラリは波形の「表示」には優れているが、以下の理由から本アプリでは採用しない：
- **直接編集との統合が困難**: WaveDromが生成するSVGは読み取り専用であり、各波形セグメントにインタラクティブなイベントを付与する仕組みがない
- **レンダリングの制御が限定的**: セル単位のホバー/選択状態の視覚的フィードバック、カーソル表示、ドラッグ操作中のリアルタイム反映など、エディタに必要なUI要素をWaveDromの外から制御するのが難しい
- **再レンダリングコスト**: 1セルの変更のたびにWaveDrom全体を再レンダリングする必要があり、大規模波形でのインタラクション性能が劣化する

#### 自前SVGレンダリングの方針
- React SVGコンポーネントとして波形を描画
- 各タイムステップの波形セグメントが独立したインタラクティブなSVG要素
- WaveDromと同等の見た目（波形の形状・色・ラベル）を再現
- セル単位の仮想DOM差分更新により、部分的な再レンダリングで高速動作
- 将来的にWaveDromのSVG出力と比較して品質を検証可能

### 3.4 デプロイ
- 静的ファイルとしてホスティング可能（GitHub Pages, Netlify 等）
- オフラインでもローカルで `index.html` を開いて使用可能（目標）

## 4. 機能要件

### 4.1 波形のGUI編集（コア機能）

#### 4.1.1 信号の管理

| 機能 | 説明 |
| :--- | :--- |
| **信号の追加** | 新しい信号行を追加する |
| **信号の削除** | 既存の信号行を削除する |
| **信号の並べ替え** | ドラッグ＆ドロップで信号の順序を変更する |
| **信号名の編集** | 信号名をインライン編集する |
| **信号のグループ化** | WaveDromのグループ構文に対応（ラベル付きグループ） |

#### 4.1.2 波形の編集
各信号行はグリッド（タイムステップ）単位で状態を編集できる。

| 波形状態 | WaveDrom記法 | 説明 |
| :--- | :--- | :--- |
| **Low** | `0` | 論理0 |
| **High** | `1` | 論理1 |
| **Positive Edge Clock** | `p` | 立ち上がりエッジクロック |
| **Negative Edge Clock** | `n` | 立ち下がりエッジクロック |
| **High-Z** | `z` | ハイインピーダンス |
| **Undefined** | `x` | 不定値 |
| **Data (色付き)** | `=`, `2`〜`9` | データバス（ラベル付与可能） |
| **Gap** | `\|` | 波形の省略（ギャップ） |
| **Continue** | `.` | 前の状態を継続 |

**編集操作（レンダリングされた波形上で直接実行）:**
- **クリック**: 波形上のタイムステップ位置をクリックして、現在選択中のツール（状態）を適用。波形がその場で即座に変化する
- **ドラッグ**: 波形上を横方向にドラッグし、複数タイムステップに一括で状態を適用。**ドラッグ範囲の左端セルのみ選択ツールの値（`0`/`1`/`=` など）を設定し、それより右のセルはすべて継続 `.` を挿入する**ことで自然な波形継続を実現する
- **ドラッグ中のフィードバック**: ドラッグ操作中は以下の視覚的フィードバックを表示する
  - SVG のカーソルが `grabbing` に変化し、ドラッグ操作中であることを示す
  - 開始セル（value を書き込むセル）を **黄色の明るいオーバーレイ** でハイライト
  - 継続セル（`.` を書き込むセル）を **薄い黄色のオーバーレイ** でハイライト
  - 開始・継続セルの波形ラインを明度アップして編集対象を強調
- **右クリック / ダブルクリック**: データ状態（`=`, `2`〜`9`）のセルでデータラベルをインライン編集する。クリックしたセルの位置に直接テキスト入力欄が表示され（`<foreignObject>` によるSVG内入力）、Enter または外側クリックで確定、Escape でキャンセル。継続 `.` セルも親のデータセルとして扱い編集可能。
- **ホバー**: マウスカーソル位置のタイムステップをハイライト表示し、編集対象を視覚的に示す

#### 4.1.3 タイムステップの管理

| 機能 | 説明 |
| :--- | :--- |
| **タイムステップ数の変更** | 全信号のタイムステップ数を増減 |
| **タイムステップの挿入** | 指定位置にタイムステップを挿入 |
| **タイムステップの削除** | 指定位置のタイムステップを削除 |

#### 4.1.4 エッジリレーション（矢印）
- WaveDromの `edge` 構文に対応
- 信号間の因果関係（セットアップ/ホールド、遅延など）を矢印で表現
- GUIから矢印の始点・終点を指定して追加・削除

### 4.2 インタラクティブ波形キャンバス（統合型エディタ）
編集とレンダリングを分離せず、レンダリングされた波形そのものが編集UIとなる統合型のアプローチを採る。

- 自前のReact SVGコンポーネントにより波形を描画
- 描画された波形の各タイムステップが直接クリック/ドラッグ可能
- 編集操作に対して波形が即座に視覚的に変化（リアルタイムフィードバック）
- ホバー時のハイライト、選択中ツールに応じたカーソル変化
- グリッドライン（タイムステップ境界）を薄く表示し、編集位置を明確に
- **ドラッグ中**: 操作範囲をオーバーレイ色でリアルタイムに視覚化（value セルは明るい黄色、継続セルは薄い黄色）。カーソルも `grabbing` に変更

#### 波形の描画仕様

| 波形状態 | SVG表現 |
| :--- | :--- |
| **Low (`0`)** | 下端の水平線 |
| **High (`1`)** | 上端の水平線 |
| **Posedge Clock (`p`)** | 立ち上がり→立ち下がりの矩形波 |
| **Negedge Clock (`n`)** | 立ち下がり→立ち上がりの矩形波 |
| **High-Z (`z`)** | 中央の水平線（破線） |
| **Undefined (`x`)** | 左端 `<`・右端 `>` のスパイク付き六角形領域（赤色ハッチング） |
| **Data (`=`, `2`〜`9`)** | 左端 `<`・右端 `>` のスパイク付き六角形領域（色付き塗りつぶし）＋中央ラベル |
| **Gap (`\|`)** | 波線による切断表示 |
| **Continue (`.`)** | 前の状態を境界線なしで延長 |

#### 継続（`.`）の描画ルール

- **`1.` `0.`**: 境界に区切り線なし・フラットに延長（`1 1` との違いが明確）
- **`11` `00`（明示的な同値繰り返し）**: セル境界で中間高さまでディップ／バンプして戻り、信号変化を可視化
- **`=.` Data継続**: 左端 `<` なし・右端 `>` なし（矩形塗りつぶし）で連続表示
- **`x.` Undefined継続**: 左端スパイクなし・矩形ハッチングで延長

#### 左右スパイクの制御ルール（x / Data 共通）

| セルの状況 | 左端 `<` | 右端 `>` | 塗りつぶし形状 |
| :--- | :--- | :--- | :--- |
| 単独セル（前後が異なる状態） | あり | あり | 六角形 |
| 先頭セル（次が `.`） | あり | なし | 左 `<` 付き五角形 |
| 継続セル（前が `.`、次は別状態） | なし | あり | 右 `>` 付き五角形 |
| 中間継続セル（前後ともに `.`） | なし | なし | 矩形 |

### 4.3 WaveDrom JSONエディタ（サイドパネル）
- 波形キャンバスと並べて、WaveDrom JSONのテキストエディタをサイドパネルとして表示
- GUI操作 → JSON自動更新（単方向同期）
- JSONの直接編集 → 波形キャンバス自動更新（双方向同期）
- JSONの構文エラー時はエラー表示し、最後の有効な状態を維持

### 4.4 ファイル入出力

| 機能 | 説明 |
| :--- | :--- |
| **新規作成** | 空の波形データで開始 |
| **ファイルを開く** | `.json` ファイルを読み込み（WaveDrom JSON形式） |
| **名前を付けて保存** | WaveDrom JSON形式で `.json` ファイルとしてダウンロード |
| **SVGエクスポート** | レンダリング結果をSVGファイルとしてダウンロード |
| **PNGエクスポート** | レンダリング結果をPNGファイルとしてダウンロード |

### 4.5 ヘッダー・フッター設定
- WaveDromの `head` / `foot` 構文に対応
- タイトル（`text`）の編集
- `tick` / `tock`（サイクル番号表示）の設定

### 4.6 コンフィグ設定
- WaveDromの `config` 構文に対応
- `hscale`（水平スケール）の変更

### 4.7 Undo / Redo
- 編集操作の取り消し / やり直し
- キーボードショートカット対応: `Ctrl+Z` / `Ctrl+Shift+Z`

## 5. UI構成

### 5.1 全体レイアウト
波形の「プレビュー」と「編集」を分離せず、レンダリングされた波形を直接編集する統合型レイアウトを採用する。

```text
┌─────────────────────────────────────────────────────┐
│  ツールバー                                          │
│  [新規][開く][保存][SVG][PNG] [Undo][Redo]           │
│  ツール: [0][1][p][n][z][x][=][色] [+step][-step]   │
├─────────────────────────────────────┬───────────────┤
│                                     │               │
│  波形キャンバス（統合型エディタ）       │  JSONエディタ  │
│                                     │  パネル        │
│  ┌─信号名─┬──波形（直接編集可能）──┐  │  （開閉可能）  │
│  │ clk    │ ┌┐┌┐┌┐┌┐┌┐┌┐┌┐┌┐  │  │               │
│  │ req    │ ___╱‾‾‾‾╲_______   │  │  {             │
│  │ ack    │ _________╱‾‾╲___   │  │   "signal":.. │
│  │ data   │ xxxx<D0><D1>xxxx   │  │  }             │
│  └────────┴────────────────────┘  │               │
│  [+信号追加]                        │               │
├─────────────────────────────────────┴───────────────┤
│  ステータスバー: カーソル位置、選択ツール、信号数等      │
└─────────────────────────────────────────────────────┘
```

### 5.2 ツールバー
- ファイル操作ボタン群（新規、開く、保存、SVG/PNGエクスポート）
- Undo / Redo ボタン
- 波形状態ツールパレット: 現在のペンツールを選択
  - `0`(Low), `1`(High), `p`(PosClk), `n`(NegClk), `z`(Hi-Z), `x`(Undefined)
  - `=`(Data) + データ色セレクタ（`2`〜`9`に対応）
  - `.`(Continue), `|`(Gap)
- タイムステップ増減ボタン
- ズームコントロール

### 5.3 波形キャンバス（統合型エディタ）
アプリのメイン領域。レンダリングされた波形がそのまま編集UIとなる。

- **信号ラベル領域（左側固定）**
  - 信号名を表示（ダブルクリックでインライン編集）
  - ドラッグ＆ドロップで信号の並べ替え
  - 右クリックでコンテキストメニュー（削除、複製、グループ化 等）
- **波形描画領域（右側、横スクロール可能）**
  - 自前SVGによる波形レンダリング
  - 各タイムステップのセグメントがインタラクティブ（クリック/ドラッグ/ホバー対応）
  - グリッドライン（タイムステップ境界）を薄く表示
  - ヘッダー行にタイムステップ番号またはtick/tockラベルを表示
  - エッジリレーション（矢印）を波形上にオーバーレイ描画
- **信号追加ボタン（下部）**

#### 編集のインタラクション詳細

| 操作 | 動作 |
| :--- | :--- |
| **波形上をクリック** | クリック位置のタイムステップに選択中ツールの状態を適用 |
| **波形上をドラッグ** | ドラッグ範囲のタイムステップに一括で状態を適用 |
| **ホバー** | 対象タイムステップをハイライト。ツール適用後のプレビューを半透明で表示 |
| **データセルをダブルクリック** | ラベル編集用インラインポップアップを表示 |
| **信号名をダブルクリック** | 信号名のインライン編集 |
| **信号名をドラッグ** | 信号の並べ替え |
| **右クリック** | コンテキストメニュー（信号の削除/複製/グループ化、タイムステップ挿入/削除 等） |
| **ホイールスクロール** | 横方向: タイムステップ方向スクロール |
| **Ctrl + ホイール** | ズームイン/アウト |

### 5.4 JSONエディタパネル（サイドパネル）
- 波形キャンバスの右側に表示
- 開閉トグルボタンで表示/非表示を切り替え（デフォルト: 非表示で波形を広く使える）
- シンタックスハイライト付きテキストエディタ
- 波形キャンバスと双方向同期

### 5.5 ステータスバー
- 現在のカーソル位置（信号名 + タイムステップ番号）
- 選択中のツール名
- 信号数 × タイムステップ数

## 6. データモデル
内部データはWaveDrom JSON形式をそのまま使用する。

### 6.1 WaveDrom JSON 形式（例）
```json
{
  "signal": [
    { "name": "clk",   "wave": "p........" },
    { "name": "req",   "wave": "0.1..0..." },
    { "name": "ack",   "wave": "0...1..0." },
    { "name": "data",  "wave": "x...=.=.x", "data": ["D0", "D1"] },
    {},
    ["Bus Interface",
      { "name": "addr",  "wave": "x..=.=.x.", "data": ["A0", "A1"] },
      { "name": "wdata", "wave": "x...=.=.x", "data": ["W0", "W1"] }
    ]
  ],
  "edge": [
    "req~>ack", "ack~>data"
  ],
  "head": {
    "text": "Read Transaction"
  },
  "config": {
    "hscale": 2
  }
}
```

### 6.2 アプリ内部状態
```typescript
interface AppState {
  /** WaveDrom JSON データ（信号、エッジ、head/foot/config を含む） */
  waveformData: WaveDromData;

  /** Undo/Redo 用の履歴スタック */
  undoStack: WaveDromData[];
  redoStack: WaveDromData[];

  /** UI状態 */
  selectedTool: WaveTool;        // 現在選択中の編集ツール
  selectedSignal: number | null; // 選択中の信号インデックス
  jsonPanelVisible: boolean;     // JSONパネルの表示/非表示
}
```

## 8. 非機能要件

| 項目 | 要件 |
| :--- | :--- |
| **パフォーマンス** | 100信号 × 50タイムステップでスムーズに動作 |
